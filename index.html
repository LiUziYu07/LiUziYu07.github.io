第五章

指令格式

操作码（OP） + 地址码字段（A）

  • 操作码：指明指令的操作特性与功能
  • 地址码：协助表示操作数或操作数的地址

操作码分定长与变长两种，操作码的数量取决于计算机指令的数量:\(L_{OP} = [log_2N]\)

根据一条指令中地址码的个数：

  • 三操作数指令

  • 二操作数指令

    根据指向的数据存储位置不同分为

    • RR（寄存器、寄存器）—— 原操作数和目的操作数均使用寄存器存放
    • RS（寄存器、存储器）——～和～一个放在寄存器中，一个放在主存
    • SS（存储器、存储器）——～～两个操作均放在主存中

  • 一操作数指令

  • 0操作数指令

指令长度根据计算机字长划分

  • 半字长指令
  • 单字长指令
  • 双字长指令

寻址方式

寻找指令或操作数的方法

指令的寻址方式：

  • 顺序寻址

    程序计数器PC+1

  • 跳跃寻址

    转移指令

操作数的寻址方式：

操作数来源：

  1. 指令地址字段
  2. 寄存器
  3. 主存

有效地址：操作数在内存中的地址，一般用E表示

操作数寻址方式：形成操作数有效地址的方法

  • 隐含寻址

    ADD

  • 立即数寻址

    ADD AX, 2038H

  • 直接寻址

    MOV AX, [2038H]

  • 间接寻址

    给出的地址中存储的是操作数所在内存单元的地址

  • 相对寻址

    将PC的值和指令中的地址相加，作为操作数的地址

  • 基址寻址

    将基址寄存器的内容的值和指令中的地址D相加，作为操作数的地址

  • 变址寻址

    将变址寄存器的内容的值和指令中的地址D相加，作为操作数的地址

  • 堆栈寻址

    破坏性独处：读数据的同时也离开了堆栈

  • 存储器堆栈

    • 入栈：

      Msp <----- (A) SP <----- SP - 1

    • 出栈：

      SP <----- SP + 1 (A) <------Msp

##CISC和RISC

CISC

  1. 指令系统复杂，指令数量多，寻址方式多，指令格式多
  2. 采用微程序控制
  3. 各种指令都可以访问存储器
  4. 有专用寄存器
  5. 难以优化编译

RISC

  1. 选取使用频率最高的一些简单指令
  2. 指令系统简化，长度固定
  3. 只有取/存数访问存储器
  4. 以流水线方式工作
  5. 不用或少用微程序
  6. 优化编译

MIPS

两类寄存器：

  1. 通用寄存器

    32个

  2. 特殊寄存器

    PC、HI、L0

寻址方式

  1. 寄存器寻址
  2. 基址或偏移寻址
  3. 立即数寻址
  4. PC相对寻址
  5. 伪直接寻址

三类指令

R型：只有寄存器寻址

  • 双目运算，rs，rt为第一和第二源操作数，rd指明存放结果的寄存器
  • 移位运算，对rt的内容进行移位，所移位数由shamt字段指定

I型：立即数寻址、基址（偏移寻址）、相对寻址、寄存器寻址

  • 双目运算，rs和立即数字段的值作为第一和第二操作数放入rt中
  • Load/Store指令，将rs和立即数字段的值经过符号位扩展后相加作为Load/Store的存储单元地址
  • 分支指令，将rs和rt寄存器中的数据进行操作码规定的运算，根据结果跳转。转移的目标采用相对寻址，将PC的内容和立即数字段经符号扩展后的内容相加

J型：伪直接寻址

  • 无条件跳转，目标地址由PC高四位与26位直接地址经2位后的值拼接得到

##主存的数据存放

存储字长和数据字长

  • 数据字长：计算机一次能处理的二进制数的位数
  • 存储字长：主存的一个存储单元所包含二进制位数

大多数计算机采用字节编址

大端和小端数据存放方式

大端：存储器的低字节地址单元存放数据的最高字节

小端：存储器的低字节地址单元存放数据的最低字节

字节编址的存储按边界对齐数据的存放方法

假设数据字长32位，存储字长64位

  • 双字数据的起始地址最末三位为000（8字节的整数倍）
  • 单字数据的起始地址的最末二位为00（4字节的整数倍）
  • 半字数据的起始地址的最末一位为0（2字节的整数倍）

主存结构及其工作过程

CPU与存储交换信息时，先给出地址，该地址经过地址译码驱动器后，选中存储体中对应的存储单元，然后由控制线路控制写入或读出

存储分级的理论基础：局部性原理

时间局部性：现在被访问的指令在不久的将来还会被再次访问

空间局部性：现在访问一个指令，下一次访问的指令会大概率在当前指令附近

SRAM存储器结构

  • 存储体：存储单元的集合
  • 地址译码器：把二进制表示的地址转换成输出电位，驱动相应的读写电路时，选择相应的读写单元
  • 驱动器：
  • I/O电路：控制选中单元的读/写和信号放大
  • 片选与读/写控制电路：用于存储器的扩容
  • 输出驱动电路：用于多片输出的互连或与双向数据总线的连接

单译码和双译码

不同的译码结构，与存储体当中存储单元的排列有关

存储单元中有行、列地址，只有行列都被选中，该单元才会被选中

双译码：将来自CPU的地址线，分行与列分别送到两个译码器进行译码

双译码的优点：可以大大减少译码输出线

###DRAM的刷新操作

原因：保存信息的电容上电荷维持的时间有限，一般为2ms

以下讨论：

刷新周期为2ms，存储矩阵采用128*128结构，存储体的读写周期为0.5\(\mu s\)

三种刷新方式：

  1. 集中式刷新

    2000/0.5 = 4000个读写时间段，前3872个读写时间段用于读写，后128读写时间段用于刷新，这段时间称为死时间

    存储器平均读写周期 \[ \overline{T} = 2ms/(4000 - 128) = 0.5165\mu s \]

  2. 分散刷新

    每个存储周期分成两部分，前一部分为读写时间段，后一部分为刷新时间段

  3. 异步刷新

    将2ms分成128（存储器个数）个时间段，每段时间为2000/128，再将每段时间分成两部分，前一部分用于读写，后一部分剩余0.5\(\mu s\)用于刷新

如果存储器是由若干单体构成时，以单体的容量为计算的依据，同时考虑双译码的原则

刷新地址由专门的器件————刷新地址寄存器组成

字、位扩展

存储器需要连接的线：

  • 地址线：地址线的数量与CPU要访问的主存空间有关
  • 数据线：数据线的数量与计算机字长有关

  • RAM控制线包括片选和读写控制，ROM只有片选信号

  • 位扩展：数据位不够，即存储体不够大

  • 字扩展：空间不足，存储体不够多

    扩展之后还是一次只能访问一个存储体

    • 线选法

      片选信号直接由地址产生，无法充分利用存储空间，地址空间离散（因为片选不能同时选中多个）

    • 全译码法

      片选信号经译码产生，地址连续切不重叠

    • 部分译码法

      片选信号由部分地址译码产生，地址空间连续单存在重叠（有多个地址指向同一存储体），因为存在不参与译码的地址线

存储器接口

  • 8位存储器

  • 16位存储器

    使用两个8位存储器

    \(A_0\)和\(\overline{BHE}（高字节允许）\)配合实现，两者均为0时为表示访问从偶地址开始的两个字节单元，两者均为1时备用

    偶地址字：一次访问

    奇地址字：两次访问

  • 32位存储器

    使用四个8位存储器

    \(A_0\)和\(A_1\)产生\(\overline{BE_3}\)~\(\overline{BE_0}\)

    访问32位数据时，\(\overline{BE_3}\)~\(\overline{BE_0}\)全部取0

    访问16位数据时，\(\overline{BE3} / \overline{BE2}\) 或 \(\overline{BE1} / \overline{BE0}\)

    访问单字节，选择相应的\(\overline{BE_i}\)为0

  • 64位存储器

    使用八个8位存储器

    \(A_0,A_1,A_2\)和\(\overline{BE7}\)~\(\overline{BE0}\)配合实现

    访问64位数据，\(\overline{BE_7}\)~\(\overline{BE_0}\)全部取0

    访问32位数据，\(\overline{BE_3}\)~\(\overline{BE_0}\)和\(A_2\)来选择

    访问8位数据，数据仅通过\(A_3\)~\(A_0\)

###并行存储器

双端口存储器

使用两组相互独立的读写电路，同时读写时需要使用一个判断逻辑BUSY来延时一个读写控制器的工作，每组读写电路各有一个判断逻辑

多体交叉存储器

由\(2^n\)个容量相同的存储器组成，各有自己的读写线路、地址寄存器、数据寄存器

高位多体交叉

类似存储器字扩展

高位段地址选择不同存储体（产生片选信号），低位段地址选择一个存储体内的某一存储单元

不同存储体的地址不相邻

同一存储体内地址相邻

适用于共享存储器的多机系统

#####低位多体交叉

与高位多体交叉相反

高位段地址选择存储体内的某一存储单元，低位段地址经过译码产生片选信号，选择不同存储体

相邻地址分布在不同存储体中，同一存储体中地址不相邻

各模块读写将重叠进行

可在一个存储周期内存取多个字

计算存储器带宽、多体交叉总线周期

####高速缓冲存储器Cache

程序访问的局部性原理
  • 时间局部性：针对一个数据的多次访问
  • 空间局部性：针对一个数据块中的不同数据

#####Cache的组织及访问Cache流程

CPU与Cache之间交换信息的基本单位是字

Cache与主存之间信息交换的单位是块

Cache的一个块包含若干个字，称为行

相联存储器是一种根据内容访问的存储器

Cache中的主存格式地址

主存块地址+块内偏移地址

主存块地址分为：

标记和索引

  • 主存块地址

    主存分块，CPU访问主存按块划分得到的地址，用于标记CPU所访问的主存单元所在的主存块号

  • 块内偏移地址

    CPU所要访问的单元在某块内的偏移值

  • 索引

    指出CPU访问Cache存储体的范围

  • 标记

    判断CPU要访问的内容是否在Cache中的依据

三种Cache映射方式

地址映射：将主存地址变换为Cache地址

地址映射：CPU访问主存，主存地址变换为Cache地址，从而查找Cache

  • 全相连映射

    主存分块，Cache分行，包含相同数量的字

    主存的一块可被映射到Cache的任意行

    主存地址分为：块号、块内地址

  • 直接映射

    主存按Cache容量分区，区内分块

    Cache分行，主存分区内块数和Cache行数相同

    主存地址分为：主存区号，区内块号，块内地址

  • 组相联映射

    Cache分组，每组中的行数相同

    主存分组，每组中的块的数量与Cache的组数量相同

    Cache每组中包含k行称为k路组相联

Cache的命中率

\(N_c\)表示Cache完成存取的总次数，\(N_m\)表示主存完成出去的总次数，\(h\)定义为命中率 \[ h = \frac{N_c}{N_c+N_m} \] 用\(t_c\)表示命中时Cache的访问时间，\(t_m\)表示未命中时的主存访问时间，Cache/主存系统的平均访问时间为: \[ t_a = ht_c + (1 - h)t_m \] 设\(r = \frac{t_m}{t_c}\)表示主存慢于Cache的倍率，e表示访问效率: \[ e = \frac{t_c}{t_a} = \frac{1}{h + (1 - h)r} \]

Cache的一致性问题

  • Cache的脏位

    表示Cache中的数据是否被修改过

    一般用于Cache的写回策略

  • Cache的有效位

    表示有不经过Cache的主存更新（如DMA），此时Cache中数据无效

Cache的结构为：

Tag + Data + Valid + Dirty

虚拟存储器

三种虚拟存储器

  • 段式

    按程序的自然段作为主存和辅存之间交换的信息单位

    使用段表来登记段的调入情况，段表存储在主存

    保护了用户程序的相对独立性

    管理复杂，容易造成主存碎片

    将程序分成固定长度的页作为主存与辅存之间交换的信息单位

    使用页表来登记页的调入情况，页表存储在主存

    管理方便，主存空间浪费少

    破坏了源程序的相对独立性

  • 段页式

    对程序段分页，页作为主存和辅存之间交换的信息单位，但共享和保护按段进行

    同时使用段表和页表

    集合了页式和段式的优点

运算方法

定点数的加减法运算以及溢出判断

####补码 加减法运算

加法：

\([X+Y]_{补} = [X]_补 + [Y]_补\)

减法：

\([X-Y]_补 = [X]_补 + [-Y]_补\)

求\([Y]_补\)技巧：直接对\([Y]_补\)连同符号位在内取反，然后在最低位加1

基本器件：全加器FA

补码运算结果的溢出判断

  • 方法1

    检测操作数和运算结果的符号位：

    \(V = X_0Y_0\overline{S_0}+\overline{X_0}\overline{Y_0}S_0\)

    \(V\)得1时表示运算结果溢出

  • 方法2

    运算时最高有效位的进位为\(C_1\)，符号位的进位为\(C_0\)，则溢出检测电路为：

    \(V=C_0\otimes C_1\)

  • 方法3

    采用变形补码：

    正数符号位为00，负数符号位为11

    \([X]_补 =X_{f1}X_{f2}.X_1...X_n\)

    上溢出：双符号位为01

    下溢出：双符号位为10

    溢出判断：\(V = X_{f1}\otimes X_{f2}\)

移码的加减法运算

加法：

\([X+Y]_补 = [X]_移 + [Y]_补\)

减法：

\([X-Y]_补 = [X]_移 + [-Y]_补\)

####移码的溢出判断

采用双符号位判断

移码的双符号位：第一个符号位为0，第二个符号位正数为0，负数为1

当运算结果的第一个符号位为0时，无溢出

运算结果的第一个符号位为1时，溢出

####补码的一位乘法运算方法

在运算开始时，\(Y_{n+1}\)补成0

算法：

  • 如果\(Y_{n+1} = Y_{n}\)，部分积加0，并右移一位结果
  • 如果\(Y_{n+1} < Y_n\)，部分积加\([-X]_补\)，并右移一位结果
  • 如果\(Y_{n+1} > Y_n\)，部分积加\([X]_补\)，并右移一位结果

  • 对阶

    移动阶码小的浮点数的尾数，使两个数的阶码相同

    对齐的步骤

    • 求阶差\(\Delta E\)
    • 将阶码小的那个浮点数的尾数向右移动\(\Delta E\),并同时将其阶码加1

  • 尾数运算

    用补码加减法法则，计算尾数

  • 结果的规格化

    • 左规格化：尾数结果为\(00.0\Phi...\Phi\)和\(11.1\Phi...\Phi\),尾数向左移动1位且阶码减1，直到尾数形式为：\(00.1\Phi...\Phi\)或\(11.0\Phi...\Phi\)
    • 右规格化：尾数的结果为\(01.\Phi...\Phi\)或\(10.\Phi...\Phi\)，尾数向右移动且阶码加1，直到尾数形式为:\(00.1\Phi...\Phi\)或\(11.0\Phi...\Phi\)

  • 舍入

    0舍1入：如果向右移出的是1，则在最低位加1；否则，不处理

    恒置1:只要数字位1被移掉，就将最后一位恒置为1

  • 溢出处理

    阶码上溢：阶码的符号位为01

    阶码下溢：阶码的符号位为10

####乘除法

阶码的运算采用补码/移码的加法法则

尾数的运算采用补码的乘法法则

第二章

计算机中的数据表示方法

原码

数值位不变

符号位0表示正数，1表示负数 ，又称符号-数值表示法

原码表示的二进制小数无法表示1与-1，有正0和负0之分

反码

符号位用0表示正数，1表示负数

数值位，正数与正值相等，负数反码的数值位是真值的数值位按位取反

反码表示的二进制小数无法表示1与-1，有正0和负0之分

补码

符号位用0表示正数，1表示负数

数值位正数补码与真值相同，负数补码是真值的数值位按位取反，并在最低位加1

补码只有正0，因此补码表示的二进制小数可以表示-1

移码

移码多用于浮点数的阶码表示，其一般定义为： \[ [X]_移 = 2^n + X \] 数值位与X的补码相同，符号为与X的补码相反

做题时，阶码有几位，注意是否包含符号位

浮点数

格式：

E：阶码位数，决定数据的范围

M：尾数位数，决定数的精度

单精度：32位，S（1位符号位），E（8位），M（23位）

双精度：64位，S（1位符号位），E（11位），M（52位）

E和M的取值对IEEE754浮点数的意义：

  • \(E = 0，M = 0\)，表示机器0
  • \(E = 0，M \neq 0\)，\(N = (-1)^s * 2^{-126}*0.M\) 表示非规格化的浮点数
  • $ 1 E 254 \(，\)N = (-1)^s2^{E-127}1.M$,表示规格化的浮点数
  • \(E = 255, M = 0\)，表示无穷大，对应\(\frac{x}{0}\)
  • \(E = 255, M \neq 0\)，\(N = NaN\)，表示Not a Number，表示一个非数值，\(\frac{0}{0}\)

尾数的规格化：尾数对应的十进制数大于0.5，这样就可以少记一位1

补码的规格化：

  • 正数规格化后，尾数形式为：0.1XX...X
  • 负数规格化后，尾数形式位：1.0XX...0

校验码

码距：将一组编码中任何两个合法编码之间编码的不同最小位数称为码距

码距d与校验码的检错e、纠错t能力的关系如下：

  • \(d \geq e + 1\) 可检e位错误
  • \(d \geq 2t + 1\) 可检t位错误
  • \(d \geq e+t+1\) 可检验e个错误，并纠正t个错误 \((e \geq t)\)

奇检验

待传数据中1的个数为奇数，包括检验位

偶检验

待传数据中的1个个数为偶数，包括检验位

奇偶检验的特点，不能检验偶数个错误

可通过行列交叉奇/偶检验，检错和改错

海明检验

具有检测和纠正错误的一种编码

将传输的信息按规律分成若干组，每组安排一个检验位

假设有r位检验位，数据位共k位，共表示\(2^r\)种不同状态，用一种状态表示无差错，剩余表示错误，满足 \[ 2^r - 1\geq k + r \] 位数为2的幂的码用作检验，其余位置填入数据位，每个数值位\(i\)由下标相加等于\(i\)的检验位所检验

指错字为r位，排列为\(G_nG_{n-1}...G_1\)，其值表示哪一位出错，其中\(G_i\)等于 \[ G_i = P_i \otimes P_i 检验的数值位 \]

计算机性能指标

基本性能指标

字长

指一次参与运算的数据的基本长度，用二进制位数的长度来衡量

一般与计算机内部的寄存器位数，数据总线，运算器，存储器字长有关

早期的计算机字长一般16位（如8086，8080），现在一般32位或64位

存储容量

存放二进制信息的总位数

主存容量：

  • 存储单元数X存储字长（MAR反应存储单元个数，MDR反应存储字长）
  • 字节数

辅寸容量：

  • 字节数

运算速度

时钟周期：节拍周期或T周期，为时钟频率的倒数，处理操作最基本的时间单位

\(CPI\)：执行每条指令所需要的平均周期数

CPU执行时间：CPU计算某个任务所花费的时间，不包括IO

响应时间：计算机完成一个任务所花费的全部时间，包括存储访问、IO、磁盘访问，用于衡量计算机执行单个任务时的性能

吞吐量：计算机系统在单位时间内完成的总工作量，是计算机系统执行多个任务时的性能表现

\(CPI\):执行每条指令（一个程序中的所有指令）所需要的平均时钟周期数

\(CPI_i\):执行每条指令（一类指令中的所有指令）所需要的平均时钟周期数

每类指令的条数\(IC_i\)，程序中总指令条数设为\(IC\)，\(CPI\)和\(CPI_i\)的关系为 \[ CPI = \displaystyle \sum^{n}_{i = 1}CPI_i *P_i = \sum^{n}_{i = 1}CPI_i*\frac{IC_i}{IC} \] CPU执行时间计算方法： \[ CPU执行时间 = CPU 时钟周期总数 * 时钟周期 \\ =(指令条数*CPI)*时钟周期 \]

CPU的性能与计算机体系结构的关系：

（1） 时钟频率反映了计算机的实现技术和生产工艺

（2）\(CPI\) 反映了计算机的实现技术和指令集结构

（3）\(IC\)反映了计算机指令系统的设计和编译技术

\(MIPS\) 指CPU每秒执行的百万条机器指令数，计算公式为： \[ MIPS =\displaystyle \frac{时钟频率}{CPI*10^6} = \frac{指令数}{程序执行时间*10^6} \] \(MFLOPS\)指CPU每秒执行的百万条浮点操作次数，计算公式为: \[ MFLOPS = \frac{程序中的浮点操作次数}{程序执行时间*10^6} \]
