第五章

指令格式

操作码（OP） + 地址码字段（A）

  • 操作码：指明指令的操作特性与功能
  • 地址码：协助表示操作数或操作数的地址

操作码分定长与变长两种，操作码的数量取决于计算机指令的数量:\(L_{OP} = [log_2N]\)

根据一条指令中地址码的个数：

  • 三操作数指令

  • 二操作数指令

    根据指向的数据存储位置不同分为

    • RR（寄存器、寄存器）—— 原操作数和目的操作数均使用寄存器存放
    • RS（寄存器、存储器）——～和～一个放在寄存器中，一个放在主存
    • SS（存储器、存储器）——～～两个操作均放在主存中

  • 一操作数指令

  • 0操作数指令

指令长度根据计算机字长划分

  • 半字长指令
  • 单字长指令
  • 双字长指令

寻址方式

寻找指令或操作数的方法

指令的寻址方式：

  • 顺序寻址

    程序计数器PC+1

  • 跳跃寻址

    转移指令

操作数的寻址方式：

操作数来源：

  1. 指令地址字段
  2. 寄存器
  3. 主存

有效地址：操作数在内存中的地址，一般用E表示

操作数寻址方式：形成操作数有效地址的方法

  • 隐含寻址

    ADD

  • 立即数寻址

    ADD AX, 2038H

  • 直接寻址

    MOV AX, [2038H]

  • 间接寻址

    给出的地址中存储的是操作数所在内存单元的地址

  • 相对寻址

    将PC的值和指令中的地址相加，作为操作数的地址

  • 基址寻址

    将基址寄存器的内容的值和指令中的地址D相加，作为操作数的地址

  • 变址寻址

    将变址寄存器的内容的值和指令中的地址D相加，作为操作数的地址

  • 堆栈寻址

    破坏性独处：读数据的同时也离开了堆栈

  • 存储器堆栈

    • 入栈：

      Msp <----- (A) SP <----- SP - 1

    • 出栈：

      SP <----- SP + 1 (A) <------Msp

##CISC和RISC

CISC

  1. 指令系统复杂，指令数量多，寻址方式多，指令格式多
  2. 采用微程序控制
  3. 各种指令都可以访问存储器
  4. 有专用寄存器
  5. 难以优化编译

RISC

  1. 选取使用频率最高的一些简单指令
  2. 指令系统简化，长度固定
  3. 只有取/存数访问存储器
  4. 以流水线方式工作
  5. 不用或少用微程序
  6. 优化编译

MIPS

两类寄存器：

  1. 通用寄存器

    32个

  2. 特殊寄存器

    PC、HI、L0

寻址方式

  1. 寄存器寻址
  2. 基址或偏移寻址
  3. 立即数寻址
  4. PC相对寻址
  5. 伪直接寻址

三类指令

R型：只有寄存器寻址

  • 双目运算，rs，rt为第一和第二源操作数，rd指明存放结果的寄存器
  • 移位运算，对rt的内容进行移位，所移位数由shamt字段指定

I型：立即数寻址、基址（偏移寻址）、相对寻址、寄存器寻址

  • 双目运算，rs和立即数字段的值作为第一和第二操作数放入rt中
  • Load/Store指令，将rs和立即数字段的值经过符号位扩展后相加作为Load/Store的存储单元地址
  • 分支指令，将rs和rt寄存器中的数据进行操作码规定的运算，根据结果跳转。转移的目标采用相对寻址，将PC的内容和立即数字段经符号扩展后的内容相加

J型：伪直接寻址

  • 无条件跳转，目标地址由PC高四位与26位直接地址经2位后的值拼接得到
